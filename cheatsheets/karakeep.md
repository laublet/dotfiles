## Daily workflow

### Capture (several times / day)

1. Extension **Karakeep** on the page (or paste URL in web UI).
2. Pick list (`read-later` or specific).
3. Optional tag `wiki-candidate` for Obsidian pipeline.
4. Do **not** add to Firefox bookmarks (except Daily shortcuts).

### Triage (2вҖ“3Г— / week, ~10 min)

1. Open http://karakeep.home.loicaublet.fr
2. Open list **`read-later`** (or `dev`, etc.).
3. For each item: open вҶ’ read вҶ’ **archive/delete** OR keep вҶ’ retag/list.

### Promote to knowledge (when ready)

| Goal | Action |
|------|--------|
| Long article to keep as text | Web Clipper вҶ’ `Literature/Clippings/` |
| Idea / note in Main | Zettelkasten or `0 - Main Notes` |
| Research wiki page | Clip/capture in **Research** вҶ’ skill `ingest-clipping` вҶ’ `Wiki/` |
| Obsidian reminder | Tag `wiki-candidate` вҶ’ sync (below) |

### Obsidian sync (вүҲ1Г— / week)

Vault **Main** only:

1. Command palette вҶ’ **Hoarder Sync: Start Sync**
2. Read `Literature/Karakeep/` (only `wiki-candidate` by default)
3. When done: remove tag or archive in Karakeep

```bash
just hoarder-sync-setup      # install / update plugin (Main)
just link                    # community-plugins-main.json
```

API key: Karakeep вҶ’ Settings вҶ’ API Keys вҶ’ paste in plugin settings (never commit `data.json`).

## Troubleshooting

| Problem | Fix |
|---------|-----|
| Firefox bookmarks refilled | `just firefox-bookmarks-clean`; check Sync вҶ’ Bookmarks off |
| Empty `Literature/Karakeep/` | Normal if nothing tagged `wiki-candidate`; or clear **Included tags** in plugin settings |
| Sync fails from Obsidian | On home LAN / Tailscale; check API URL + key; `useObsidianRequest: true` |
| Duplicate noise in Research | Do not enable hoarder-sync there вҖ” Main only |
